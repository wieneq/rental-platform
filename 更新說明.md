# 系統更新說明

## ✅ 已完成更新（2026-03-22）

### 認證系統改版

**從 Email 登入 → 改為 帳號密碼登入**

#### 主要變更

1. **登入方式**
   - ❌ 舊：Email + 密碼
   - ✅ 新：帳號 + 密碼

2. **註冊流程**
   - ❌ 舊：填寫 Email、密碼、姓名、角色
   - ✅ 新：填寫帳號、密碼、姓名、角色
   - ✅ **註冊即啟用，無需審核**

3. **帳號格式**
   - 帳號長度：至少 3 個字元
   - 可使用：英文、數字
   - 範例：`admin`, `owner1`, `renter123`

#### 新的測試帳號

| 帳號 | 密碼 | 角色 |
|------|------|------|
| **admin** | demo123 | 管理員 |
| **owner1** | demo123 | 器材擁有者 |
| **owner2** | demo123 | 器材擁有者 |
| **renter1** | demo123 | 租用者 |
| **renter2** | demo123 | 租用者 |

#### 使用說明

**登入**
1. 訪問 http://localhost:3000
2. 點擊「登入」
3. 輸入帳號（如：`admin`）和密碼（`demo123`）
4. 點擊「登入」按鈕

**註冊新帳號**
1. 點擊「立即註冊」
2. 填寫資料：
   - 帳號（至少 3 個字元）
   - 密碼（至少 6 個字元）
   - 姓名
   - 選擇角色：租用者 或 器材擁有者
3. 點擊「註冊」
4. **立即啟用，無需等待審核**
5. 自動登入並導向控制台

#### 技術細節

**修改的檔案：**
- `api/data/seed.js` - 範例資料改用 username
- `api/controllers/authController.js` - 登入/註冊邏輯更新
- `public/login.html` - 登入表單改為帳號輸入
- `public/register.html` - 註冊表單調整順序
- `api/controllers/contractController.js` - PDF 租約顯示帳號

**資料庫結構變更：**
```javascript
// 舊結構
{
  id: 1,
  email: 'admin@rental.com',
  password: '...',
  name: '系統管理員',
  role: 'admin'
}

// 新結構
{
  id: 1,
  username: 'admin',
  password: '...',
  name: '系統管理員',
  role: 'admin'
}
```

#### 優點

✅ **更簡單** - 不需要填寫 Email 格式  
✅ **更快速** - 帳號可以更短、更好記  
✅ **免審核** - 註冊後立即可用  
✅ **適合展示** - 快速建立測試帳號  

#### 注意事項

⚠️ 展示版使用記憶體儲存，重啟後所有新註冊的帳號會清空  
⚠️ 預設測試帳號會在每次啟動時自動載入  
⚠️ 帳號不可重複（系統會檢查）
